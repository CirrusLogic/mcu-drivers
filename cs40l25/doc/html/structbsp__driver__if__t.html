Perform a Double-Buffered ("db") I2C Write.

This will first write the contents of write_buffer_0 to the I2C device, and then write the contents of write_buffer_1.

Parameters
[in]bsp_dev_idID of the I2C device corresponding to the I2C peripheral to reset
[in]write_buffer_0pointer to array of first batch of bytes to write
[in]write_length_0total number of bytes in write_buffer_0
[in]write_buffer_1pointer to array of second batch of bytes to write
[in]write_length_1total number of bytes in write_buffer_1
[in]cbpointer to callback function
[in]cb_argpointer to argument to use when calling callback
Returns
  • BSP_STATUS_FAIL if bsp_dev_id is invalid, if any portion of I2C transaction failed
  • BSP_STATUS_OK otherwise

◆ i2c_read_repeated_start

uint32_t(* bsp_driver_if_t::i2c_read_repeated_start) (uint32_t bsp_dev_id, uint8_t *write_buffer, uint32_t write_length, uint8_t *read_buffer, uint32_t read_length, bsp_callback_t cb, void *cb_arg)

Perform an I2C Write-Repeated Start-Read transaction.

This is the common way to read data from an I2C device with a register file, since the address of the register to read must first be written to the device before reading any contents.

Perform transaction in the order:

  1. I2C Start
  2. I2C write of write_length bytes from write_buffer
  3. I2C Repeated Start
  4. I2C read of read_length bytes into read_buffer
  5. I2C Stop

BSP will decode bsp_dev_id to the correct I2C bus and I2C address.

Parameters
[in]bsp_dev_idID of the I2C device corresponding to the I2C peripheral to reset
[in]write_bufferpointer to array of bytes to write
[in]write_lengthtotal number of bytes in write_buffer
[in]read_bufferpointer to array of bytes to load with I2C bytes read
[in]read_lengthtotal number of bytes to read into read_buffer
[in]cbpointer to callback function
[in]cb_argpointer to argument to use when calling callback
Returns
  • BSP_STATUS_FAIL if bsp_dev_id is invalid, if any portion of I2C transaction failed
  • BSP_STATUS_OK otherwise
